# Conventions for AI Agents

## Build entrypoint

- `./compile.sh <command> BOARD=... BRANCH=... RELEASE=...` — single entrypoint.
- First arg is the command; everything else is `KEY=VALUE` (any order).
- Always run `./compile.sh requirements` first to install host deps.
- Common commands: `image`, `kernel`, `kernel-config`, `uboot-config`, `kernel-patch`, `docker`, `flash`.

## Kernel commit pinning

- Kernel pinned via `KERNELBRANCH="commit:<sha>"` in `config/sources/families/include/meson64_common.inc:49`
- To update: `git ls-remote https://github.com/leftymods/linux-6.18.y.git main` → update SHA in that file
- Also update the fetch in `lib/functions/compilation/kernel-git-oras.sh:207-228` if the ref format changes
- Custom kernel sources (not `kernel.org` or `googlesource.com`) are cloned directly, bypassing ORAS bundles

## Auto-generated files — DO NOT edit

- `lib/library-functions.sh` — generated by `lib/tools/gen-library.sh`
- `Dockerfile` — generated; use `./compile.sh docker` instead
- `config/boards/*.conf` status: `.conf`=supported, `.csc`=community, `.wip`=wip, `.tvb`=tv-box, `.eos`=end-of-life
- `.github/CODEOWNERS` — generated by `.github/generate_CODEOWNERS.sh`

## Deprecated workflows (no `KERNEL_CONFIGURE=yes` etc.)

Use the dedicated commands instead:
- `./compile.sh kernel-config BOARD=... BRANCH=...` (not `KERNEL_CONFIGURE=yes`)
- `./compile.sh uboot-config BOARD=... BRANCH=...` (not `UBOOT_CONFIGURE=yes`)
- `./compile.sh kernel-patch BOARD=... BRANCH=...` (not `CREATE_PATCHES=yes`)

## Renovate version tracking

Pinned tool versions in `lib/*.sh` are tracked by Renovate's regex manager via the pattern `${NAME_VERSION:-<semver>} # https://github.com/owner/repo/releases`. Allow-listed names: SHELLFMT, SHELLCHECK, ORAS, BATCAT. Follow the format in `renovate.json5:30-88` exactly or Renovate silently drops tracking.

## Dependencies

- Python tools pinned in `requirements.txt` (fixed versions, updated by Dependabot).
- Pre-commit hooks: gitleaks, shellcheck, cpplint, pylint, end-of-file-fixer, trailing-whitespace.
- Workspace path must not contain whitespace.

## Board support

- **One board**: `AtriStation` (Amlogic S905X3, meson-sm1 family) — `config/boards/atristation.conf`
- Board config enables: RTL8822CS WiFi/BT, TAS5825M audio amp, rotary encoder, AtriOS sound stack

## Patches policy

- All board-specific changes go directly into the kernel fork (`leftymods/linux-6.18.y`), NOT as patch files.
- **ALL kernel patch files** in `patch/kernel/archive/meson64-*` are disabled (`.patch.disabled`). Never enable them.
- **ALL rtl8723cs staging driver patches** are skipped on `meson64` (in `drivers_network.sh`). No other driver patches should be applied.
- Only exception: Armbian's built-in DT/overlay auto-patching is OK (it's part of the build pipeline, not patch files).
- If a patch file needs to be removed/deleted, use `git mv` or at minimum do NOT `git add` the whole directory — only stage specific files.

## CI

- Main build: `.github/workflows/build-atrios.yml` → composite action in `action.yml`
- Docker image: `.github/workflows/build-docker.yml` — pushed on Dockerfile/requirements.txt changes to main
- Kernel ORAS bundle: `.github/workflows/build-oras.yml` — weekly and manual dispatch
